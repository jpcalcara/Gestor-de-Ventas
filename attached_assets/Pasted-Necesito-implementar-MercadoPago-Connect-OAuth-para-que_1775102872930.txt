Necesito implementar MercadoPago Connect (OAuth) para que cada negocio (business) pueda conectar su propia cuenta de MercadoPago. Los vendedores de ese negocio usan las credenciales del negocio para procesar pagos. El flujo es el estándar de marketplace/plataforma de MP.

---

CONTEXTO DEL PROYECTO

- Multi-tenant: businesses → branches → users (roles: sistemas/admin/vendedor)
- Schema: tabla businesses, tabla branches (con businessId FK), tabla users
- Stack: React 18 + TanStack Query v5 + shadcn/ui / Node + Express + Drizzle ORM + PostgreSQL
- MercadoPago SDK ya instalado (mercadopago npm package)
- paymentMethodEnum ya existe: ["efectivo", "debito", "credito", "qr", "transferencia"]

3. BACKEND – Endpoints OAuth

GET /api/mercadopago/connect
- requireAuth, solo roles "admin" o "sistemas"
- Construye la URL de autorización de MercadoPago:
  https://auth.mercadopago.com/authorization?client_id={MP_APP_ID}&response_type=code&platform_id=mp&redirect_uri={MP_REDIRECT_URI}&state={businessId}
- El campo state debe ser el businessId del admin logueado (req.session.businessId)
- Retorna: { authUrl: string }

GET /api/mercadopago/callback
- Ruta pública (es el redirect de MP, no requiere sesión)
- Recibe query params: code, state (que contiene el businessId)
- Llama a la API de MP para intercambiar el code por tokens:
  POST https://api.mercadopago.com/oauth/token
  Body: { client_id: MP_APP_ID, client_secret: MP_CLIENT_SECRET, code, grant_type: "authorization_code", redirect_uri: MP_REDIRECT_URI }
- Guarda en la tabla businesses: mpAccessToken, mpRefreshToken, mpUserId, mpPublicKey, mpConnectedAt, mpExpiresAt
- Redirige al frontend a /settings?mp=connected en caso de éxito, /settings?mp=error en caso de fallo

DELETE /api/mercadopago/disconnect
- requireAuth, solo admin/sistemas
- Limpia todas las columnas mp* del business del admin logueado poniéndolas en null
- Retorna: { success: true }

GET /api/mercadopago/status
- requireAuth
- Retorna si el business del usuario logueado tiene MP conectado:
  { connected: boolean, mpUserId: string | null, connectedAt: string | null }
- NUNCA retornar el access token ni el refresh token al frontend

5. BACKEND – Endpoints de pago usando credenciales del negocio

POST /api/mercadopago/create-preference
- requireAuth
- Obtiene businessId desde req.session.businessId
- Usa getMPClientForBusiness(businessId) para procesar con las credenciales del NEGOCIO
- Recibe body: { items: CartItem[], paymentMethod: 'debito' | 'credito' | 'qr' }
- Para debito/credito: crea preferencia de pago estándar de MP
- Para qr: usa la API de QR instore de MP con el access token del negocio
- Retorna: { preferenceId, initPoint, qrData } (qrData solo para QR)

GET /api/mercadopago/payment-status/:paymentId
- requireAuth
- Usa getMPClientForBusiness con el businessId del usuario logueado
- Retorna: { status: 'pending' | 'approved' | 'rejected' }

POST /api/sale-orders/:orderId/transfer-voucher
- requireAuth
- Recibe multipart/form-data con campo "image" (archivo de imagen)
- Usar multer para manejar el upload (instalar si no está)
- Guardar la imagen en /public/vouchers/ con nombre único (uuid + extensión original)
- Actualizar la columna transferVoucherUrl de la sale_order correspondiente
- Validar que la sale_order pertenezca a la branch del usuario logueado
- Retorna: { voucherUrl: string }

FLUJO COMPLETO ESPERADO:

1. El dueño del sistema crea su App en developers.mercadopago.com (tipo Marketplace) y obtiene MP_APP_ID y MP_CLIENT_SECRET
2. El admin de cada negocio va a Configuración → hace click en "Conectar MercadoPago"
3. Se redirige al flujo OAuth de MercadoPago donde el admin autoriza el acceso
4. MercadoPago redirige de vuelta al sistema con un code, el backend lo intercambia por tokens y los guarda en la DB del negocio
5. A partir de ese momento, cuando un vendedor de ese negocio registra una venta y elige débito, crédito o QR, el sistema usa automáticamente las credenciales MP del negocio
6. El vendedor nunca ve ni maneja ninguna credencial, solo elige el método de pago y confirma
7. El cobro se acredita directamente en la cuenta de MercadoPago del negocio
