# Prompt para Replit Agent — Cambio de plan con prorrateo + límite multisucursal

## Contexto del proyecto

SaaS multi-tenant de gestión de ventas. Stack: Node.js + Express + TypeScript (ESM) + Drizzle ORM + PostgreSQL (Neon) + React 18 + TanStack Query v5 + shadcn/ui + Tailwind + Wouter. Roles: `sistemas` (superadmin), `admin` (por negocio), `vendedor`.

Las siguientes tablas y columnas ya existen en el schema:
- `businesses`: tiene `planId`, `mpSubscriptionId`, `subscriptionStatus`, `lastPaymentAt`, `nextPaymentAt`, `gracePeriodDays`, `plan`
- `plans`: tiene `id`, `slug`, `name`, `price`, `isActive`
- `plan_features`: tiene `planId`, `featureKey`, `enabled`, `limit`
- `subscription_events`: tiene `businessId`, `type`, `description`, `amount`, `metadata`
- El SDK de Mercado Pago ya está instalado e inicializado en `server/mercadopago.ts`
- El helper `checkFeatureLimit(businessId, featureKey, currentCount)` ya existe en `server/features.ts`
- El helper `invalidateFeatureCache(businessId)` ya existe en `server/features.ts`

---

## REGLAS DE NEGOCIO

**Ciclo de facturación:** 30 días corridos desde `lastPaymentAt`. No mes calendario.

**Upgrade (plan más caro):**
- Cobrar diferencial prorrateado de inmediato via Mercado Pago (pago único, no suscripción).
- Fórmula: `total = Math.round(((precioNuevo - precioActual) / 30) * diasRestantes)`
- `diasRestantes = Math.max(0, 30 - Math.floor((Date.now() - lastPaymentAt) / 86400000))`
- El plan se activa inmediatamente al confirmar el pago (webhook MP).
- Al próximo ciclo MP ya cobra el precio nuevo completo.

**Downgrade (plan más barato) o cambio a Free:**
- NO se aplica de inmediato. NO se devuelve dinero.
- Se guarda `pendingPlanId` en el negocio.
- Se aplica automáticamente cuando MP procesa la próxima renovación (webhook pago aprobado).
- El cliente sigue con las features del plan actual hasta que vence el ciclo.
- Si el destino es Free: cancelar suscripción en MP además de guardar `pendingPlanId`.

**Sistemas cambia plan manualmente:** cambio directo sin cobro ni prorrateo.

## PARTE 2 — Backend: endpoint preview de cambio de plan

En `server/routes.ts`, agregar dentro de `registerRoutes`:

```
GET /api/subscription/change-plan/preview
  - requireAuth, solo rol admin
  - Query param: ?planSlug=pro
  - Lógica:
    1. Obtener negocio del admin (req.session.businessId)
    2. Obtener plan actual del negocio (JOIN con plans por planId)
    3. Obtener plan destino por planSlug
    4. Si no existe el plan destino → 404
    5. Calcular diasTranscurridos = Math.floor((Date.now() - lastPaymentAt.getTime()) / 86400000)
    6. diasRestantes = Math.max(0, 30 - diasTranscurridos)
    7. diferencial = Math.round(((precioNuevo - precioActual) / 30) * diasRestantes)
    8. tipo = precioNuevo > precioActual ? "upgrade" : precioNuevo < precioActual ? "downgrade" : "same"
  - Respuesta:
    {
      currentPlan: { name, price, slug },
      targetPlan: { name, price, slug },
      diasTranscurridos,
      diasRestantes,
      diferencial,
      tipo: "upgrade" | "downgrade" | "same",
      efectivaEn: null | nextPaymentAt.toISOString()
    }
```

## PARTE 3 — Backend: endpoint de cambio de plan

En `server/routes.ts`, agregar:

```
POST /api/subscription/change-plan
  - requireAuth, solo rol admin
  - Body: { planSlug: string }
  - Lógica:

    1. Obtener negocio del admin (req.session.businessId)
    2. Obtener plan actual del negocio
    3. Obtener plan destino por planSlug — si no existe devolver 400
    4. Si planSlug === plan actual → devolver 400 "Ya estás en este plan"

    5. Calcular diasRestantes igual que en el preview

    6. diferencial = Math.round(((precioNuevo - precioActual) / 30) * diasRestantes)

    === CASO UPGRADE (diferencial > 0) ===
    7a. Crear un pago único en Mercado Pago por el monto diferencial usando la Payment API:
        POST https://api.mercadopago.com/v1/payments
        Headers: Authorization: Bearer {MP_ACCESS_TOKEN}
        Body:
        {
          transaction_amount: diferencial,
          description: `Upgrade de ${planActual.name} a ${planDestino.name} - ${diasRestantes} días restantes`,
          payment_method_id: "account_money",
          payer: { email: req.session.userEmail },
          external_reference: `upgrade_${businessId}_${planDestino.slug}`,
          metadata: { businessId, fromPlan: planActual.slug, toPlan: planDestino.slug, type: "plan_upgrade" }
        }
    7b. Si MP devuelve init_point o payment_url → devolver { type: "upgrade", checkoutUrl, diferencial, diasRestantes }
    7c. Si MP devuelve status "approved" directamente (pago con saldo MP) → aplicar el upgrade inmediatamente:
        - updateBusiness(businessId, { planId: planDestino.id, plan: planDestino.slug, pendingPlanId: null })
        - createSubscriptionEvent({ businessId, type: "plan_upgrade_applied", description: `...`, amount: diferencial })
        - invalidateFeatureCache(businessId)
        - devolver { type: "upgrade_immediate", message: "Plan actualizado exitosamente" }

    === CASO DOWNGRADE O FREE (diferencial <= 0) ===
    8a. Si el destino es Free (precio === 0) Y hay mpSubscriptionId:
        - Cancelar la suscripción en MP usando cancelMPSubscription del archivo mp-subscriptions.ts
    8b. Guardar pendingPlanId = planDestino.id en el negocio
    8c. createSubscriptionEvent({
          businessId,
          type: "plan_downgrade_scheduled",
          description: `Cambio programado de ${planActual.name} a ${planDestino.name}`,
          metadata: JSON.stringify({ fromPlan: planActual.slug, toPlan: planDestino.slug, effectiveDate: nextPaymentAt })
        })
    8d. devolver {
          type: "downgrade",
          message: `El cambio se aplicará el ${nextPaymentAt.toLocaleDateString('es-AR')}`,
          effectiveDate: nextPaymentAt
        }
```

```
PATCH /api/admin/businesses/:id/plan
  - requireSistemas
  - Body: { planSlug: string }

    1. Buscar el plan por slug — si no existe devolver 404
    2. Obtener el negocio por params.id — si no existe devolver 404

         metadata: JSON.stringify({ changedBy: req.session.userId })
       })
    5. invalidateFeatureCache(params.id)
    6. devolver { success: true, plan }
```

## PARTE 6 — Backend: aplicar downgrade pendiente en webhook

```typescript
// Aplicar downgrade pendiente si existe
if (business.pendingPlanId) {
  const [pendingPlan] = await db.select().from(plans).where(eq(plans.id, business.pendingPlanId));
  if (pendingPlan) {
    await storage.updateBusiness(business.id, {
      planId: business.pendingPlanId,
      plan: pendingPlan.slug,
      pendingPlanId: null,
    } as any);
    await storage.createSubscriptionEvent({
      businessId: business.id,
      type: "plan_downgrade_applied",

    } as any);
    invalidateFeatureCache(business.id);
  }
}
```

## PARTE 7 — Backend: límite de multisucursal al crear sucursal

En `server/routes.ts`, en el endpoint `POST /api/branches`, ANTES de crear la sucursal, agregar la verificación del límite del plan para el rol `admin`:

```typescript
// Solo verificar límite para admins (sistemas no tiene límite)
if (userRole === "admin" && businessId) {
  // Contar sucursales activas del negocio
  const existingBranches = await storage.getBranchesByBusiness(businessId);
  const activeBranchCount = existingBranches.filter(b => b.isActive).length;

  // Verificar límite del plan
  const { allowed, limit } = await checkFeatureLimit(businessId, "multisucursal", activeBranchCount);
  if (!allowed) {
    return res.status(403).json({
      message: `Tu plan actual permite hasta ${limit} sucursal${limit === 1 ? '' : 'es'} activa${limit === 1 ? '' : 's'}. Actualizá tu plan para agregar más.`,
      code: "FEATURE_LIMIT_REACHED",
      feature: "multisucursal",
      limit,
      current: activeBranchCount,
    });
  }
}
```

Si el método `getBranchesByBusiness(businessId)` no existe en `server/storage.ts`, crearlo:

```typescript
async getBranchesByBusiness(businessId: string): Promise<Branch[]> {
  return await db.select().from(branches).where(eq(branches.businessId, businessId));
}
```

```typescript
const changePlanMutation = useMutation({
  mutationFn: async (planSlug: string) => {
    const res = await apiRequest("POST", "/api/subscription/change-plan", { planSlug });
    return res;
  },
  onSuccess: (data) => {
    if (data.checkoutUrl) {
      window.location.href = data.checkoutUrl;
```
